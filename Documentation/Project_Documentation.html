🏬 Project Overview

How to Generate the HTML Coverage Report

# Step 1 — Run tests with coverage collection
cd Backend\Graduation_Project_Backend.Tests
dotnet test --collect:"XPlat Code Coverage" --settings coverage.runsettings

# Step 2 — Generate HTML report (PowerShell)
$xml = Get-ChildItem "TestResults" -Filter "coverage.cobertura.xml" -Recurse `
       | Sort-Object LastWriteTime -Descending | Select-Object -First 1 -ExpandProperty FullName

reportgenerator -reports:"$xml" `
                -targetdir:"..\..\CoverageReport" `
                -reporttypes:Html

# Step 3 — Open in browser
Start-Process "..\..\CoverageReport\index.html"

How to Import and Run in Postman

1

Open Postman and Import Collection

Click Import → drag and drop MallLoyalty_Postman_Collection.json
Then import the environment file MallLoyalty_Postman_Environment.json

2

Set Environment Variables

Select the Mall Loyalty System environment. Update mallId and managerId with actual GUIDs from your database. The Base URL is already set to http://localhost:5239

3

Start the API Server

In Visual Studio: Press F5 to run the project
Or in terminal: cd Backend && dotnet run

4

Run Collection

Click Run Collection → Select all folders → Click Run Mall Loyalty System
Or run individual folders by right-clicking each one.

⚠️

Important: Run the Auth folder first — it sets the sessionId used by all other requests. Also update the mallId and managerId environment variables to match your database.

Run from Command Line (Newman)

# Install Newman
npm install -g newman

# Run the full collection
newman run Documentation\MallLoyalty_Postman_Collection.json \
       -e Documentation\MallLoyalty_Postman_Environment.json \
       --reporters cli,html \
       --reporter-html-export Documentation\newman-report.html

🏗️ Architecture

The project follows a clean layered architecture with clear separation of concerns.

Backend/ ├── Controllers/ ← HTTP layer (thin, no business logic) │ ├── AuthController.cs │ ├── StoresController.cs │ ├── AnnouncementsController.cs │ ├── OffersController.cs │ ├── CouponsController.cs │ ├── TransactionsController.cs │ ├── DashboardController.cs │ ├── ChatbotController.cs │ ├── RealtimeController.cs │ └── UserInfoController.cs │ ├── Service/ ← Business logic (tested by unit tests) │ ├── AuthService.cs │ ├── StoresService.cs │ ├── AnnouncementsService.cs │ ├── OffersService.cs │ ├── RewardsService.cs │ ├── DashboardService.cs │ ├── ChatbotService.cs │ ├── Auth/ PhoneNumberService.cs, PasswordHasher │ ├── Common/ UserAccessService.cs, ApiExceptions.cs │ ├── Session/ SessionService.cs │ └── Realtime/ UserPointsUpdatesService.cs │ ├── Models/ ← Entity definitions │ ├── Entities/ Mall, Store, Offer, Coupon, Transaction… │ └── User/ UserProfile, UserSession, Manager… │ ├── DTOs/ ← Request/Response data shapes ├── Data/ ← AppDbContext (EF Core) ├── Filters/ ← SessionRequiredFilter, Exception handling └── Extensions/ ← Service registration helpers
🏆

Key Achievement: The service layer is completely decoupled from the HTTP layer, which is why we can unit-test 100% of business logic using an in-memory database — no running server needed.
